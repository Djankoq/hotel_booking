# Проект: Система бронирования отелей

## Описание
Проект представляет собой API для системы бронирования отелей, разработанный на FastAPI. Он включает в себя аутентификацию пользователей, управление отелями, номерами и бронированиями. Особенностью проекта является рекомендательная система, которая подбирает похожие отели на основе различных параметров.

## Стек технологий
- **Backend**: Python, FastAPI
- **База данных**: PostgreSQL
- **Контейнеризация**: Docker, Docker Compose
- **Миграции**: Alembic
- **ORM**: SQLAlchemy
- **Аутентификация**: JWT (python-jose), passlib
- **Валидация данных**: Pydantic
- **Запуск приложения FastAPI**: Uvicorn 

## Как запустить проект

1. **Клонируйте репозиторий:**
   ```bash
   git clone https://github.com/Djankoq/hotel_booking.git
   cd hotel_booking
   ```

2. **Запустите проект с помощью Docker Compose:**
   ```bash
   docker-compose up --build
   ```
   Эта команда соберет образы, запустит контейнеры с приложением и базой данных, а также автоматически применит миграции.

3. **Проверьте работу API:**
   Интерактивная документация (Swagger) будет доступна по адресу:
   [http://localhost:8000/docs](http://localhost:8000/docs)

## Вспомогательные скрипты

Для удобства тестирования и разработки в проекте есть два скрипта:

- **Заполнение базы данных тестовыми данными:**
  ```bash
  docker exec -it hotel_app_container python -m scripts.seed
  ```
  Этот скрипт наполняет базу данных сгенерированными отелями, комнатами, пользователями и бронированиями.

- **Очистка базы данных:**
  ```bash
  docker exec -it hotel_app_container python -m scripts.clear_data
  ```
  Этот скрипт удаляет все данные из таблиц, оставляя их пустыми.

## Описание рекомендательной системы

Для подбора похожих отелей используется алгоритм, основанный на **нормализованном евклидовом расстоянии**.

Поскольку сравниваемые характеристики отелей (цена, вместимость, описание, расположение) имеют разный масштаб и тип, все они приводятся к единому диапазону значений от 0 до 1 перед расчетом:

- **Расположение (`location`)**: Бинарная характеристика. `0` — если расположение совпадает, `1` — если нет.
- **Описание (`description`)**: Сходство текстов оценивается через количество общих слов. `0` — полное совпадение, `1` — нет ни одного общего слова.
- **Вместимость (`capacity`)** и **Цена (`price_per_night`)**: Числовые значения нормализуются по формуле `(x1 - x2) / (xmax - xmin)`, где `xmax` и `xmin` — максимальное и минимальное значения в выборке.

Итоговое расстояние показывает, насколько один отель "далек" от другого. Чем меньше значение, тем более похожими считаются отели.
